# Архитектура пакета

## Слои

1. `rules/` хранит исполняемую бизнес-логику в authoring-форме CLI.
2. `manifest.json` задаёт публичные pipelines и редакторские метаданные.
3. `operators/node/` содержит только недостающие доменные вычисления.
4. `jsonspecs-cli` валидирует проект, запускает samples и строит snapshot.
5. `@jsonspecs/rules` компилирует snapshot и выполняет выбранный pipeline.

Сервис-потребитель не читает отдельные файлы `rules/` и не собирает их сам. Он
загружает готовый `dist/snapshot.json`, регистрирует операторный пакет и один раз
вызывает `compileSnapshot` при старте.

## Идентичность сборки

`sourceHash` вычисляется Rules по нормативной JCS-формуле RC.7 и идентифицирует
исполняемое содержимое snapshot. `build-info.json` хранит внешние сведения:
версию проекта и runtime, exports, количество артефактов и digest операторного
пакета. Метаданные редактора не входят в `sourceHash`.

## Метаданные

`manifest.catalog` является единственным UI-каталогом:

- `fields` описывает payload/context-пути;
- `entrypoints` описывает экспортируемые сценарии;
- `artifacts` даёт название каждому артефакту полного замыкания;
- `operators` объясняет встроенные и доменные операторы.

Поля `btName`, `businessDescription`, отдельные `field-contracts` и runtime
адаптация `legacyCode` не входят в контракт версии 1.0.0.

## Расширения

Внешний оператор имеет закрытую JSON Schema и чистую `evaluate`-функцию. Пути
объявляются в стандартных `field`/`inputs`, постоянные настройки в `params`.
Доступ к целому payload, resolver, pipeline или текущему времени запрещён.
