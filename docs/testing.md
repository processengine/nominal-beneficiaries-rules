# Контракт тестирования

`npm test` выполняет пять уровней проверки:

1. `jsonspecs validate --fail-on-warning` проверяет закрытые схемы, regex,
   ссылки, объединённый DAG и полное замыкание exports.
2. `node --test tests/operators.test.js` запускает эталонные векторы доменных операторов.
3. `jsonspecs test` выполняет каждый sample и требует покрытие всех exports.
4. `jsonspecs build --fail-on-warning` создаёт snapshot и build-info.
5. `scripts/verify-package.mjs` сверяет версии, JCS-хеш, каталог, сведения о
   сборке, метаданные бизнес-требований и покрытие всех кодов ошибок примерами.

Sample содержит верхнеуровневый `pipelineId`, `payload`, `context` и точную
проекцию ожидаемого результата. При изменении issue обязательно фиксируются
его `code`, `level` и `field`; `exact: true` запрещает дополнительные ошибки.
Среди примеров обязательно есть отдельная проверка значения `null`.

Старые зафиксированные сборки и прежняя среда выполнения не входят в тестовый контур v1. История до
миграции доступна по тегу `v0.8.3`.
