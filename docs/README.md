# Документация

- [`architecture.md`](architecture.md) описывает границы пакета и поток сборки;
- [`testing.md`](testing.md) фиксирует исполняемый регрессионный контракт;
- [`requirements-traceability.md`](requirements-traceability.md) связывает ошибки правил с разделами и пунктами бизнес-требований;
- [`release-procedure.md`](release-procedure.md) задаёт обязательную публикацию.

Нормативная семантика DSL находится в `jsonspecs/spec` 1.0.0-rc.7. Этот каталог
описывает только решения и процессы конкретного пакета бенефициаров.
