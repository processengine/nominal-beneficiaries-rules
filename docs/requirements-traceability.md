# Связь правил с бизнес-требованиями

Источником для ссылок в правилах служит файл
`Бизнес-требования для работы с бенефициарами номинальных счетов.docx`.

- `SHA-256`: `521375fedfffed655736cadc6d13bd564d55aef68777cc8a3f96d24bc6235578`;
- последняя запись в истории изменений документа: версия 14 от 20 октября 2025 года;
- разделы и пункты извлекаются из структуры DOCX и текста вложенных таблиц;
- `page` — справочная страница начала раздела в текущей выгрузке;
- `pageKind: "section-start"` прямо указывает, что страница не является устойчивым идентификатором.

## Формат метаданных

Spec RC.7 не допускает верхнеуровневый `rule.meta`. Авторские сведения можно
хранить в `rule.issue.meta`. Rules v4 включает их в `sourceHash` и без изменений
возвращает в найденной ошибке.

```json
{
  "issue": {
    "level": "ERROR",
    "code": "FL.INN.LENGTH_12",
    "message": "ИНН ФЛ должен содержать ровно 12 цифр",
    "meta": {
      "requirements": [
        {
          "document": "Бизнес-требования для работы с бенефициарами номинальных счетов.docx",
          "documentSha256": "sha256:521375fedfffed655736cadc6d13bd564d55aef68777cc8a3f96d24bc6235578",
          "section": "2. Набор полей для физических лиц (ФЛ)",
          "item": "п. 1 — ИНН",
          "page": 13,
          "pageKind": "section-start"
        }
      ]
    }
  }
}
```

Для общего правила перечисляются все таблицы, к которым оно относится. Если
конкретное ограничение сохраняет действующий API-контракт, но не записано в
бизнес-требованиях отдельно, `explanation` объясняет это одним предложением.

Условия, сценарии и справочники не создают ошибок выполнения. Поэтому у них
нет `issue` и, следовательно, нет `issue.meta`; их назначение описывает
`manifest.catalog.artifacts`.

## Зафиксированные расхождения

- Бизнес-требования показывают даты как `ДД.ММ.ГГГГ`, а JSON API и правила
  используют `YYYY-MM-DD`.
- В строке ОГРНИП бизнес-требования указывают 13 цифр. Правило сохраняет
  стандартный 15-значный формат ОГРНИП действующего API.
- `$context.currentDate`, номер номинального счёта, КПП иностранного ЮЛ и часть
  детализации регистрирующего органа не выделены отдельными строками таблиц.
  Для таких правил дана ссылка на общий пункт о проверках и пояснение.

`scripts/verify-package.mjs` проверяет наличие описаний всех полей, метаданных
всех правил с ошибками, точного хэша документа и примеров для каждого кода.
